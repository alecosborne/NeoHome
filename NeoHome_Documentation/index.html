Einleitung


Zielgruppe


Es gibt zwei klare Zielgruppen für den Einsatz des Basis-Projektes. Die erste Zielgruppe der Anwender zeichnet sich dadurch ab, das sie das Beispiel mit den zur Verfügung gestellten Mitteln nutzt und auf die beim Anwender eingesetzte Version anpasst. Die zweite Gruppe ist die der Entwickler, welche Erweiterungen in Form von Widgets und Funktionen ergänzen oder ändern kann.

Zielgruppe Fähigkeiten und Kenntnisse
Anwender HTML, JavaScript (einfach)
Entwickler HTML, JavaScript, Angular, CSS

Voraussetzungen und Tools


FHEM WebSocket Implementierung. - (Ursprünglich von ntruchsess)
Mongoose Webserver - Installations- und Konfigurationsfreier Webserver zum entwickeln und debugen
Sublimetext - oder jeder andere Editor

Bemerkung: Diewerse andere JavaScript - Abhängigkeiten werden dann in den jeweiligen Modulen erwähnt.


Einrichtung


FHEM - WebSockets


Um FHEM WebSockets beizubringen sind als erstes folgende Bibliotheken auf dem RPI zu installieren.


  1. sudo cpan App::cpanminus

  2. sudo cpanm Protocol::WebSocket

  3. sudo cpanm JSON

Danach müssen die PerlModule heruntergeladen und in das entsprechenden FHEM Verzeichnis /opt/fhem/FHEM kopiert werden.


  1. 00_websocket.pm

  2. 10_websocket_json.pm

  3. 10_websocket_fhem.pm
Bemerkung: Die PerlModule liegen diesem Paket im Verzeichnis /NeoHome/setup auch bei.

In der fhem.cfg sind mit folgenden Zeilen die Websockets zu aktivieren.


fhem.cfg 
	define wsPort websocket 8080 global
	define wsPort_json websocket_json
Bemerkung: Nach dem Reboot sollte auf der Startseite folgende Meldung erscheinen 

	ERROR: 0 0

Erste Schritte

Für einen ersten Funktionstest muss wie in der Einleitung beschrieben die WebSockets im FHEM eingerichtet und dann die Konfiguration nh_config.js angepasst werden.

Neben den FHEM URLs im Bereich Fhem constants müssen in dem Abschnitt Devicelist die FHEM Devices bekannt gemacht werden.
Jetzt muss das FHEM Device nur noch mit den Modulen (Widgets) auf den Seiten verknüpft werden.
Dazu muss in den HTML-Views im Verzeichnis views/ in den entsprechenden Directiven das Attribut data-fhem-names angepasst werden.

Beispiel der nh_config.js mit allen Applikationsrelevanten Konstanten:
...
/**
 *	Application constants
 */
var APP_NAME = "NeoHome";
var APP_VERSION = "Alpha 0.5";
var APP_ROOTPATH = "NeoHome_Seed";
var APP_DEBUG = false;
var APP_CONTROLLER_DEBUG = false;
var APP_DIRECTIVE_DEBUG = false;
var APP_SERVICE_DEBUG = false;

/**
 *	Fhem constants
 */
var FHEM_URL = "http://192.168.1.24:8083/fhem";
var FHEM_OVER_WEB_SOCKET = true;
var FHEM_WEB_SOCKET_URL = "ws://192.168.1.24:8080";
var FHEM_WEB_SOCKET_SECURE_URL = "wss://192.168.1.24:8080";
var FHEM_WEB_SOCKET_SECURE = false;

...

Module und Directives

Modul - Switch


Beschreibung


Die State/Switch Directive kann als Infoelement oder aber als Schalterelement eingesetzt werden. Dazu stehen unterschiedliche Templates zur Verfügung. Bei einem Schalter Element wird bei einem Klick der FHEM Befehl set [DEVICE] [STATE] ausgeführt.

Darstellungsformen der State/Switch Directive nh-state-element:












Beispiel


Beispiel für eine State/Switch Directive nh-state-element: 
	<nh-state-element     
		data-template="list_switch"

		data-fhem-names="FHEM_DEMO_SWITCH_1"
		data-text="{{'ELEMENTTEXT.DEMO_SWTICH' | translate}}"

		data-on-state = "on"             
		data-off-state = "off"               

		data-bg-color="white-bg"  
		data-bg-color-on="red-bg"
		data-bg-color-off="blue-bg"                       

		data-image-on="images/controls/alarmanlage_ext.png"
		data-image-off="images/controls/alarmanlage_ext_open.png"     
		data-image-class-on="iconOn"
		data-image-class-off="iconOff">
	</nh-state-element>

Parameter

data-template aussehenbestimmendes Template

  • infrastructure: 'infrastructure_state_element.html'

  • tile: 'tile_state_element.html'

  • list_double_switch: 'list_double_switch_element.html'

Abhängigkeiten


Die Abhängigkeit für die Switch Directive wird in der nh_config.js definiert. Hier muss für das Switch Element die passende Bibliothek beim Aufruf der Seite nachgeladen werden.

Beispiel einer Definition der Abhängigkeit uiSwitch: 
	.state('rooms.demo_room1', {
		url: "/demo_room1",
		templateUrl: "views/demo_room1.html",
		data: { pageTitle: 'Demo room 1' },
		resolve: {
			loadPlugin: function ($ocLazyLoad) {
				return $ocLazyLoad.load([
					{
					   name: 'uiSwitch',
					   files: ['css/plugins/angular-ui-switch/angular-ui-switch.min.css','js/plugins/angular-ui-switch/angular-ui-switch.min.js']
					}
				]);
			}
		}
	})

Textresources (Translation)

Für die Textresourcenanbindung wird das Framework von Pascal Precht verwendet. Es ermöglicht das einfache einbinden von Textresourcen. Die Einbindung kann über verschiedene Wege wie z.B Filter oder Directiven erfolgen. Es gibt eine sehr gute Dokumentation in der anhand von Beispielen die Funktionsweise und Eigenschaften sehr gut erklärt werden.
